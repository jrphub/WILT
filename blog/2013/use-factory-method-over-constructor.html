Use static Factory Method over Constructor

28 September 2013

Constructor comes with Few Limitations

The advantage of Static Factory Method over Constructors are..

  1. They can have different names specific to functionality contains, unlike constructors which are always named as the class name
  2. They are not required to create a new object each time they are invoked
  3. They can return an object of any subtypeof their own type, unlike constructor
  4. They reduce the repeatablity of creating parametrized type instance

Usage

Before


// File Class.java 
public class Class {
    public Class(String s) {
        ...
    }
}

 // File AnotherClass.java 
public class AnotherClass {
    public void method() {
        Class aClass = new Class("string");
    }
}

After


// File Class.java 
public class Class {
    private Class(String s) {
        ...
    }
    public static createClass(String s) {
        return new Class(s);
    }
}

 // File AnotherClass.java 
public class AnotherClass {
    public void method() {
        Class aClass = Class.createClass("string");
    }
}
